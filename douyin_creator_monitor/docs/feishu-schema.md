# Feishu Schema Notes

## 达人基础信息表

真实表 ID 不写入可提交文档。请在本地私有配置中维护：

```text
douyin_creator_monitor/local/feishu-ids.md
```

关键字段：

- `达人昵称`
- `达人主页地址`
- `所属平台`
- `抖音UID`
- `SecUID`
- `账号ID`
- `账号简介`
- `粉丝数`
- `关注数`
- `获赞数`
- `作品数`
- `IP属地`
- `所在地区`
- `最近检查时间`
- `最近发稿时间`
- `主页采集状态`
- `账号状态`
- `主页原始数据`
- `作品表名称`
- `作品表ID`
- `作品表链接`

字段语义：

- `创建时间`: 飞书记录创建时间，不是抖音账号创建时间。
- `最近发稿时间`: 该达人最新非置顶作品的 `create_time` 转北京时间；测试时直接使用主页列表中最新作品时间。
- `所属平台`: 当前按主页 URL 域名推断，`douyin.com` 写为 `抖音`。

## 作品表规则

- 每个作品表只记录一个达人。
- 作品表名称使用达人昵称。
- 达人基础信息表通过 `作品表名称`、`作品表ID`、`作品表链接` 关联到对应作品表。

作品表关键字段：

- `抖音作品ID`
- `作品链接`
- `作品类型`
- `是否置顶`
- `作品标题`
- `原始文案`
- `话题标签`
- `封面图URL`
- `页面可见数字`
- `页面可见数字含义`
- `发布时间`
- `详情页状态`
- `视频时长秒`
- `点赞数`
- `评论数`
- `收藏数`
- `分享数`
- `通义听悟状态`
- `语音转写全文`
- `ima状态`
- `百度网盘状态`
- `本地知识库状态`
- `采集状态`
- `最近采集时间`
- `记录时间`
- `原始作品数据`

语音转写字段建议：

- `语音转写全文`: 火山引擎 ASR 识别后，再经过 `domain_glossary.json` 规则修正得到的最终正确文案。后续写入飞书、上传 IMA、本地知识库归档都使用这一份文案。

写回规则：

- 每个作品表只记录一个达人，写回时必须明确目标作品表 ID。
- 脚本默认用 `抖音作品ID` 搜索对应记录，然后更新同一条记录。
- 脚本默认只写 `语音转写全文` 一个字段，不写 ASR 原始 JSON、纠错报告、转写状态或完成时间。
- 不要用标题、发布时间或作品链接做唯一定位，避免误写到相似作品。
## IMA 归档字段

达人基础信息表已经加入 IMA 归档字段，用来把飞书达人记录和 IMA 知识库文件夹绑定起来。

字段：

- `IMA知识库名称`
- `IMA知识库ID`
- `IMA文件夹名称`
- `IMA文件夹ID`
- `IMA同步状态`

当前约定：

- `IMA同步状态 = 已映射`：该达人已在 IMA 中有对应文件夹，后续文案可直接上传到该文件夹。
- `IMA同步状态 = 待创建文件夹`：飞书里已有达人记录，但 IMA 里还没有对应文件夹。
- `IMA同步状态 = 待上传文案`：文件夹映射存在，但文案资料还没有完成上传。
- `IMA同步状态 = 异常`：IMA 查询、上传或映射校验失败，需要人工检查。

新增达人流程需要同时维护三类资产：

1. 在达人基础信息表中新建/更新达人记录。
2. 为达人创建或关联专属作品表，并回填 `作品表名称`、`作品表ID`、`作品表链接`。
3. 在 IMA「李建荣的知识库」中创建或定位同名文件夹，并回填 `IMA知识库名称`、`IMA知识库ID`、`IMA文件夹名称`、`IMA文件夹ID`、`IMA同步状态`。

当前 IMA OpenAPI 已验证支持读取知识库、定位文件夹、上传文件到指定 `IMA文件夹ID`。公开 OpenAPI 文档中暂未看到创建知识库文件夹接口；如果新增达人时 IMA 文件夹尚不存在，先在基础信息表标记为 `待创建文件夹`，待文件夹创建后再回填文件夹 ID。
